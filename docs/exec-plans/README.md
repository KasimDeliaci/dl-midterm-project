# Execution Plans

Use this folder for task-level implementation plans when a change is large enough to need one.

- `active/`: current plans.
- `completed/`: finished plans.
- `tech-debt.md`: deferred cleanup or follow-up work.

Execution plans should describe:

- objective,
- constraints,
- affected files,
- verification gates,
- rollback/fallback notes,
- final outcome.
