# Dataset Audit

This document is the canonical Sprint 1 dataset audit location for HAM10000.

Sprint 1 dataset preparation code is implemented, but the real HAM10000 files are not present
locally yet. No dataset statistics, split files, or benchmark results have been fabricated.

## Source And Access

- Primary dataset: HAM10000.
- Canonical paper: Tschandl, Rosendahl, and Kittler, 2018.
- Official source: ISIC Archive / Harvard Dataverse.
- Configured source URL: `https://doi.org/10.7910/DVN/DBW86T`.
- Expected standard benchmark size: 10,015 images.
- Classes: `akiec`, `bcc`, `bkl`, `df`, `nv`, `mel`, `vasc`.
- License note: verify and record the exact license text from the downloaded source before final reporting.

Expected local layout:

```text
data/
├── metadata/HAM10000_metadata.csv
├── raw/
│   ├── HAM10000_images_part_1/
│   └── HAM10000_images_part_2/

```

The image lookup searches recursively under `data/raw`, so equivalent extracted folder names
are acceptable as long as files are named by `image_id` with `.jpg`, `.jpeg`, or `.png`.

## Modeling Caveats

- The dataset is class-imbalanced; accuracy alone is insufficient.
- Per-class F1 and macro-F1 should be prominent in the report.
- If metadata includes multiple images of the same lesion, splits should group by lesion ID.
- This project evaluates benchmark classification, not clinical diagnosis.

## Implemented Sprint 1 Checks

`scripts/prepare_dataset.py` now:

- loads `data/metadata/HAM10000_metadata.csv` by default,
- normalizes HAM10000 `dx` labels to a canonical `label` column,
- validates labels against the configured seven-class label set,
- recursively resolves image paths under `data/raw`,
- reports duplicate image IDs, missing labels, missing image files, and lesion ID availability,
- exports `data/processed/ham10000_audited_metadata.csv` for repeatable split-only runs,
- exports `artifacts/report_assets/tables/class_distribution.csv`,
- exports `artifacts/report_assets/figures/class_distribution.png`,
- creates `data/splits/train.csv`, `data/splits/val.csv`, and `data/splits/test.csv` only when blocking audit errors are absent.

`src/dl_midterm/data/splits.py` creates `70/15/15` splits. If complete `lesion_id`
metadata is available, splitting happens at lesion-group level and leakage is checked across
train/validation/test. If grouped stratification is impossible due to small class/group counts,
the script keeps group isolation and emits a warning.
